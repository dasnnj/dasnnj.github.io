将Hexo同时部署在github和腾讯云开发者平台或Coding初级实践教程

发表于 2019-01-26 20:52:03 | 更新于 2019-02-21 22:30:55 | 分类于 tech ， Hexo | 阅读次数：
写在前面的话

其实我也是这两天才接触到Hexo，之前是用的wordpress在阿里云上挂着。觉得Hexo好像更符合现在我的审美，so, do it!

嗯前面安装git和node.js我这边就省略掉了。作为一个爱搞事的，这些东西电脑上都有

还有就是我照着网上的教程是没问题，但是走到一些页面的小功能的时候，就不起作用了，可能是版本更新不兼容了

一. 安装Hexo，初始化
  1. npm install -g hexo全局安装Hexo 创建一个文件夹如blog，不用进去（可以用hexo -v检验是否安装成功）
  2. hexo init blog 初始化这个blog和文件夹名字要一样，否则又创建个新的
  3. npm install安装所需要的依赖
  4. 后面就 hexo s -g 就是发布之前先生成静态文件 ，s:server，g:generate，访问下localhost:4000看ok不（不起作用，提示什么hexo <commands>什么东西了，就进到blog的目录下，使用hexo命令）
  5. 应该没有5了，如果上面没成功，那你去搜搜别人的初始化都怎么弄的，然后再回来看我剩下的实践
二. 创建github公开库

  1. 有个point就是创建Repository的名字格式是 username.github.io，（看到有的博主只用的username就行，你可以尝试一下，不行的话删了就行）比如我的是 dasnnj.github.io，是为了能生成page服务

  2. 两步，输入Repository name，然后点击 create repository 按钮

  3. 建错删除的话，点进去新建的库，点击setting，点击最下面的删除，需要输入库的名字才能确认删除

  4. 没问题的话，还是要点进去setting，往下面滑动到GitHub Pages标题下面，照着那个链接点进去，不出意外就能直接访问到你的这个repository

三. 创建腾讯云开发者平台（或Coding）公开库

  1. 项目地址格式是 username.coding.me，格式不对会404哦，项目名称随便，确定就ok

  2. 创建完记得进入代码浏览，初始化一下项目，添加一个readme文档就行了

  3. 进入page服务，然后开启

四. 配置服务并将文件部署到Github

  1. 复制上面创建的两个库的git地址

  2. 修改最下面的deploy，格式类似我这样的

    1. 执行hexo clean && hexo g && hexo s 清除缓存，生成静态文件，本地发布
    2. 页面上没问题的话，就可以执行hexo d
    3. 会弹出输入github账号密码，和腾讯开发者平台的账号密码。后面通过生成ssh私钥，公钥就不用频繁输入用户名密码,参考windows生成git公钥
    4. 部署成功，按照各自平台的pages服务提示的网址即可访问
